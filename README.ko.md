## Credence란 무엇인가

Credence는 노련한 고정수익 신용 분석가의 방법론을 AI 에이전트가 직접 로드하고 실행할 수 있는 형태로 패키지화한 것입니다. 이는 **에이전트 프레임워크나 독립 실행형 앱이 아닌**, 국제 채권 시장을 위한 기관급 신용 분석을 위한 **도메인 방법론 스킬 팩**입니다.

### 핵심 원칙

전통적인 재무 분석은 기업의 신용 위험을 재무제표에서 읽을 수 있다는 암묵적 가정에 의존합니다. 이 가정은 세 가지 업종 유형에서 체계적으로 실패합니다:

| 업종 유형 | 재무 분석 실패 이유 | 가장 중요한 요소 위치 |
|---|---|---|
| **정책 주도형**(태양광, 반도체) | 정책 주기가 산업 수요 상한선을 결정; 급격한 정책 전환은 수주일 만에 산업을 파괴 | 산업 정책/지정학(대차대조표 미기재) |
| **기술 장벽형**(첨단 장비, 바이오의약) | 핵심 자산(IP, 파이프라인, 인증)이 대차대조표에 없음; 수익 창출 전 기업은 PE/PB로 평가 불가 | 기술 로드맵/핵심 IP(대차대조표 미기재) |
| **자산 리스형**(데이터센터, 인프라 REIT) | REIT 유사 프로필; 핵심 지표는 NOI/DSCR이며 전통적 재무 지표가 아님 | 고객 리스 품질(대차대조표 미기재) |

**가장 중요한 신용 요소는 대차대조표에 절대 나타나지 않습니다.** 외부 신용 등급은 실제 신용 악화보다 평균 17개월 이상 지연됩니다(엔론, 리만 브라더스, 와이어카드, 그리스 국채 — 모두 디폴트 수개월 전까지 투자적격 등급).

### 두 가지 이론적 기반

| 이론 | 의미 | 엔진 구현 |
|---|---|---|
| **모자이크 이론** | 개별 공개 데이터 조각은 단독으로 무의미하지만 조합하면 완전한 그림을 형성 | 다중 소스 데이터 집계, 신호 스태킹, 신뢰도 가중치 |
| **정보 완전성 이론** | 데이터 갭은 결함이 아닌 위험 신호입니다. "이 데이터가 없다"는 사실 자체가 해당 차원에 불확실성이 있음을 알려줌 | 모든 분석 결론에 데이터 완전성 점수와 갭 리스트 포함 |

### 아키텍처 개요

```
시스템 인텔리전스 계층(레이어 4)
  전염 맵 x 집중도 대시보드 x 시스템 위험 지수(SRI)
                        |
                  단일 발행자 결과
                        |
              ┌─────────┴─────────┐
              │   모자이크 엔진    │   신호 추출 + 조립 + 완전성
              │   (레이어 1)      │
              └─────────┬─────────┘
                        |
           ┌────────────┼────────────┐
           │            │            │
      Track A       Track B      Track C+: 다중 이해관계자
     기본 분석      시장 가격      6개 바이사이드 역할
                     신호
           │            │
           └──────┬─────┘
                  ▼
       교차 검증 매트릭스
    합의 -> 상호 강화
    차이 -> 가장 가치 있는 통찰
                  │
                  ▼
           통합 출력
     등급 + 신호 + 완전성 보고서
```

## 엔진 아키텍처

### 레이어 1: 모자이크 엔진

**조각난 공개 데이터에서 신호 추출.**

모자이크 엔진은 전체 분석 파이프라인의 데이터 수신 계층으로 작동합니다. 비정형 다중 소스 공개 데이터를 입력받아 신뢰도 가중치가 적용된 정형 신호로 변환하여 다운스트림 이중 트랙 분석에 제공합니다.

#### 신호 추출

엔진은 7가지 공개 소스 범주(거시 정책, 산업 데이터, 공급망 가격, 기업 제출 서류, 소송/규제 기록, 채권 시장 데이터, 거시경제 지표)에서 데이터를 수집합니다. 모두 무료 공개 채널(WebSearch, SEC EDGAR, 중앙은행 포털, FRED, TRACE 등)을 통해 이루어집니다.

#### 모자이크 조립

개별 데이터 포인트는 규칙 엔진을 사용하여 산업 피라미드에 따라 조립됩니다. 조립 프로세스는 소스 신뢰성, 시간적 최근성, 교차 소스 확인에 따라 신호를 스택하여 각 발행자에 대한 구조화된 신호 맵을 생성합니다.

#### 완전성 평가

모든 분석 결론에는 정량적 **완전성 점수**(0-100)와 명시적 갭 리스트가 포함됩니다. 엔진은 다음을 구분합니다:

- **알려진 알려짐** — 여러 소스에서 확인된 데이터 포인트
- **알려진 알려지지 않음** — 식별되어 위험 신호로 플래그 지정된 데이터 갭
- **알려지지 않은 알려지지 않음** — 방법론적 한계로 문서화된 구조적 사각지대

#### 주요 지표

| 지표 | 설명 |
|---|---|
| 완전성 점수 | 분석 차원별 0-100점 |
| 신호 신뢰도 | 소스 가중치가 적용된 각 추출 신호의 신뢰도 |
| 갭 영향 | 각 데이터 갭이 등급 신뢰도에 미치는 영향에 대한 정성적 평가 |

### 레이어 2: 이중 트랙 엔진

**기본 분석(Track A)과 시장 가격 신호(Track B) 교차 검증.**

이중 트랙 엔진은 두 개의 독립적인 분석 트랙을 병렬로 실행한 후 출력을 교차 검증합니다. 트랙 간의 차이가 엔진이 생성하는 가장 가치 있는 통찰을 만들어냅니다.

#### Track A: 기본 분석

산업 피라미드 프레임워크(10차원 점수 시스템 D1-D10)를 적용하여 다음을 평가합니다:

| 차원 | 초점 |
|---|---|
| D1-D3 | 구조적 산업 위치, 정책 환경, 경쟁 우위 |
| D4-D6 | 비즈니스 모델 회복력, 수익 안정성, 비용 구조 |
| D7-D8 | 재무 정책, 자본 구조, 유동성 |
| D9-D10 | 거버넌스, 경영진 실적, 외부 지원 |

점수는 엄격한 레이어별 진행을 따릅니다: 레이어 1(가장 중요한 구조적 요소)이 통과해야 레이어 2 분석이 의미를 가집니다. 레이어를 건너뛸 수 없습니다. 재무 레이어(L4)는 상위 레이어 판단을 검증하는 데 사용됩니다.

#### Track B: 시장 가격 신호

4단계 시장 내재 신호를 분석합니다:

| 신호 단계 | 지표 |
|---|---|
| 신용 스프레드 | Z-스프레드, 자산스왑 스프레드, CDS 프리미엄 |
| 변동성 | 가격 변동성, 내재 변동성 스큐 |
| 자금 흐름 | 프라이머리 시장 수요, 세컨더리 회전율, 투자자 구성 |
| 등급 이동 | 외부 등급 추세, 전망 변경, 워치리스트 항목 |

#### 교차 검증 매트릭스

| Track A / Track B | 긍정 신호 | 부정 신호 |
|---|---|---|
| **긍정** | **상호 강화** — 높은 신뢰도 | **Track B 선행** — 시장 가격이 기본보다 먼저 반응 |
| **부정** | **Track A 선행** — 기본이 시장 가격보다 먼저 악화 | **상호 확인** — 위기의 높은 확실성 |

충돌 시, **Track A(감사 가능한 공개 재무 사실)이 우선**하며 외부 등급보다 우선합니다.

#### 등급 매핑

엔진의 내부 0-10 복합 점수는 18노치 스케일로 3대 국제 신용평가기관에 매핑됩니다——**점수가 높을수록 등급이 높습니다**(9.5-10.0 = AAA가 최상위, 0-0.9 = D가 최하위). 일표 거부권(one-vote veto) 발동 시 복합 등급 상한은 CCC로 고정됩니다. S&P / Moody's / Fitch와의 단계별 정렬은 `dev/engine/dual-track-methodology.md` §6을 유일한 출처로 하며 여기에는 복제하지 않습니다.

**재무 프레임워크**: IFRS(국제재무보고기준)와 US GAAP(일반적으로 인정된 회계 원칙)을 모두 지원합니다. 발행자의 제출 서류에서 보고 기준을 자동 감지하고 주요 차이점(운용 리스 자본화, R&D 자본화, 이연 법인세 인식)을 조정합니다.

### 레이어 3: 다중 이해관계자 엔진

**6개 바이사이드 역할과 교차 역할 분석 매트릭스.**

다양한 시장 참여자는 동일한 신용을 다른 렌즈를 통해 봅니다. 다중 이해관계자 엔진은 6개 바이사이드 역할에 걸쳐 병렬 평가를 실행한 후 합의와 차이를 강조하는 교차 비교 매트릭스를 구축합니다.

#### 6개 역할

| # | 역할 | 핵심 의사결정 | 시간 범위 | 주요 데이터 요구사항 |
|---|---|---|---|---|
| 1 | **신용 선택자** | "이 신용을 장부에 포함해야 하는가?" — 단일 발행자 등급, 디폴트 확률 | 12-36개월 | 산업 피라미드, 재무 심층 분석, LGD/회수율, 외부 지원 |
| 2 | **포트폴리오 매니저** | "이것이 최고의 위험/보상인가?" — 상대 가치, 섹터 배분 | 6-24개월 | 상대 가치 지표, 비교 분석, 커브 포지셔닝 |
| 3 | **리스크 오피서** | "집중/전염 핫스팟은 어디인가?" — 포트폴리오 위험 모니터링 | 지속적(월간 SRI + 이벤트 구동) | 집중 대시보드, 전염 매트릭스, SRI, 스트레스 테스트 |
| 4 | **트레이더** | "오늘이 행동할 날인가?" — 실행, 시장 타이밍 | 당일~2주 | L0 신호 카드, 실시간 스프레드, 유동성 조건 |
| 5 | **어드바이저** | "내 고객은 무엇을 해야 하는가?" — 배분 조언, 적합성 | 3-12개월 | 고객 위험 프로필 오버레이, L1 스냅샷, 테마별 의견 |
| 6 | **개인 투자자** | "이 채권을 보유해야 하는가?" — 개인 투자 결정 | 6-36개월 | 단순화된 L0/L1, 매수/보유/매도 신호, 평이한 위험 요약 |

#### 교차 역할 매트릭스

여러 이해관계자가 동일한 발행자를 분석할 때 엔진은 합의/차이 매트릭스를 구축합니다:

| 측면 | 합의 시나리오 | 차이 시나리오 |
|---|---|---|
| 신용 품질 | 모든 역할이 등급 방향에 동의 | 신용 선택자 강세, 리스크 오피서 약세 -> 심층 조사 |
| 위험 선호 | PM과 리스크 오피서가 집중도에 동의 | 트레이더가 단기 기회 포착, 리스크 오피서가 꼬리 위험 경고 |
| 시간 범위 | 모든 역할의 신호가 동일한 트리거 윈도우를 가리킴 | 시간 범위 차이가 만기 불일치 위험 노출 |

**방법론 검증**: 이 엔진의 방법론은 문서화된 5개의 과거 디폴트/위기 사례——리먼 브라더스, Wirecard, Valeant, 크레디트 스위스, 그리스 소버린 채무 재조정——에 적용되었습니다(`dev/engine/validation-methodology.md` §6 참조). `validation/` 디렉토리에는 2건의 공개 방법론 참조 보고서가 포함되어 있으며, 전체 테스트 아카이브는 비공개로 관리되고 요청 시 제공됩니다.

### 레이어 4: 시스템 인텔리전스 계층

**발행자 간 시스템 위험 인식 — 전염, 집중도, 조기 경고.**

시스템 인텔리전스 계층(SIL)은 최상위 집계 계층으로, 단일 발행자 분석으로는 발견할 수 없는 시스템 위험 패턴을 감지합니다. 3개의 통합 모듈과 1개의 코딩된 모니터링 엔진으로 구성됩니다.

#### 전염 매트릭스(19개 국제 산업)

글로벌 산업 분류 기준(GICS)에 기반한 완전한 19x19 산업 간 전염 강도 매트릭스로, 모든 주요 국제 경제 부문을 포괄합니다:

| # | 산업 | 주요 패러다임 | 전염 역할 |
|---|---|---|---|
| 1 | 에너지(석유 및 가스) | P1: 경기 순환형 | 슈퍼 스프레더(높은 외부 전염) |
| 2 | 화학 | P1: 경기 순환형 | 중간 전파자 |
| 3 | 금속 및 광업 | P1: 경기 순환형 | 경기 증폭기 |
| 4 | 건설 자재 | P1: 경기 순환형 | 인프라 연계 전파자 |
| 5 | 자본재 | P1: 경기 순환형 | 제조업 전염 허브 |
| 6 | 상업 서비스 | P1: 경기 순환형 | 낮은 시스템 연관성 |
| 7 | 운송 | P4: 규제 유틸리티 | 물류 전파 벡터 |
| 8 | 자동차 | P1: 경기 순환형 | 소비-산업 브리지 |
| 9 | 내구 소비재 | P1: 경기 순환형 | 수요 순환 수신자 |
| 10 | 필수 소비재 | P2: 방어형 | 방어적, 낮은 전염 |
| 11 | 소매 | P1: 경기 순환형 | 최종 수요 전파 수신자 |
| 12 | 기술 하드웨어(반도체) | P3: 성장 | 지정학적 전염 슈퍼 스프레더 |
| 13 | 소프트웨어 및 서비스 | P3: 성장 | 낮은 직접 전염, 높은 내러티브 파급 |
| 14 | 생명공학 및 제약 | P3: 성장 | 규제 충격 수신자 |
| 15 | 의료 기기 | P2: 방어형 | 낮은 경기 연동 전염 |

| 18 | 금융(은행/보험) | P5: 금융 | **시스템 슈퍼 스프레더**(가장 높은 외부 전염) |

주요 파생 지표로는 전염 계수(CFC), 전염 취약성 계수(CVC), 전염 순노출 비율(CNER)과 함께 요인별 강도 점프를 위한 스트레스 에스컬레이션 표가 포함됩니다.

#### 5차원 집중도 대시보드

5개의 독립적인 차원에서 포트폴리오 집중 위험을 평가합니다:

| 차원 | 평가 내용 |
|---|---|
| 산업 집중도 | GICS 산업별 CR3 / CR5 / HHI / MAX1 |
| 지역 집중도 | 단일 국가/지역 비중 + 취약 지역 비중 |
| 등급 집중도 | 외부 AAA 비중 + 유사 고등급 비중 |
| 만기 집중도 | 12개월 만기 비중 + 단월 피크 |
| 자금 조달 채널 집중도 | 최대 채널 비중 + 축소 상태 |

각 차원의 임계값, 신호등 밴드, 노치 영향 매핑은 `dev/engine/concentration-framework.md`를 유일한 출처로 하며 여기에는 복제하지 않습니다.

#### 시스템 위험 지수(SRI)와 4단계 온도계

SRI는 다중 소스 신호를 단일 시스템 위험 판독값으로 집계하여 4단계 온도계로 시각화됩니다:

| 온도계 단계 | SRI 범위(0-3+ 스케일) | 의미 |
|---|---|---|
| 🟢 정상 | 0.5 미만 | 시스템 위험이 정상 범위 내 |
| 🟡 주시 | 0.5 - 1.0 | 특정 섹터의 위험 상승 |
| 🟠 경계 | 1.0 - 1.8 | 광범위한 위험 축적 |
| 🔴 위험 | 1.8 이상 | 시스템 스트레스 임박 |

SRI는 연속 0-3+ 스케일을 사용하며 백분율 체계는 사용하지 않습니다. 단계 정의와 의무 조치는 `dev/engine/systemic-warning-framework.md` §3을 유일한 출처로 합니다. SRI 계산 엔진(`src/sri_calculator.py`)은 집계 알고리즘을 구현하며, 온도계 단계는 L0 신호 카드에 입력되어 4단계 파이프라인 전체의 에스컬레이션을 트리거합니다.

#### 전망 모니터링 엔진

네 번째 코딩된 엔진(`src/outlook_engine.py`)은 다음을 제공합니다:

- **12-24개월 등급 전망** 평가(방향성 확률 포함)
- **90일 워치리스트**(자동 트리거 조건 포함)
- **등급 이동 매트릭스**(과거 전환 확률 포함)
- **지속적 모니터링 트리거**(작업 경로 레지스트리를 통해 전파)

## 4단계 파이프라인

모든 신용 분석은 4단계 체인 파이프라인을 통해 진행되며, `path_id`가 각 단계 간의 연결 키입니다:

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ ① 접수   │ -> │ ② 분석   │ -> │ ③ 보고서 │ -> │ ④ QA    │
│ (라우터)  │    │ (엔진)   │    │ (빌더)   │    │ (검증자) │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

| 단계 | 이름 | 산출물 | 호스팅 스킬 | 상태 |
|---|---|---|---|---|
| S1 | **접수** | 작업 경로 시트 | `credit-analysis-router` | ✅ 제공됨 |
| S2 | **분석** | 분석 출력 | `fixed-income-credit-analysis` | ✅ 제공됨 |
| S3 | **보고서** | 납품 지시서 | `credit-report-builder` | ✅ 제공됨 |
| S4 | **QA** | QA 판정 | `credit-qa-verifier` | ✅ 제공됨 |

**S1 — 접수(라우터)**: `credit-analysis-router` 스킬은 4질문 라우팅 메커니즘을 사용하여 모호한 사용자 요청을 구체적인 작업 경로 시트로 분류합니다. 시트에는 `path_id`, 엔진 읽기 순서, 템플릿 선택, 품질 게이트 사양이 포함됩니다. 이는 모두 `dev/engine/work-path-registry.md`의 단일 진실 공급원에서 파생됩니다.

**S2 — 분석**: `fixed-income-credit-analysis` 스킬은 경로 시트의 엔진 읽기 순서에 따라 분석을 실행합니다. 4개의 유선 경로의 경우 오케스트레이터(`src/pipeline.py`)가 해당 코딩된 엔진을 직접 호출합니다:
- **WP-RO-01** -> `src/concentration_scorer.py`(5차원 집중도)
- **WP-RO-02** -> `src/contagion_engine.py`(전염 매트릭스)
- **WP-RO-03** -> `src/sri_calculator.py`(시스템 위험 지수)
- **WP-X-05** -> `src/outlook_engine.py`(전망 모니터링)

유선되지 않은 모든 경로는 엔진 문서에 따라 LLM이 오케스트레이션합니다.

**S3 — 보고서**: `credit-report-builder` 스킬은 완료된 분석을 납품 가능한 HTML 보고서로 조립합니다. 템플릿 선택(Type 1-18)은 경로 시트 사양을 따르며 L0/L1/L2 3계층 출력 시스템에 매핑됩니다:
- **L0**(신호 카드) — 5초 요약: 등급, 전망, 당일 주요 신호
- **L1**(스냅샷) — 레이더 차트와 주요 이상 목록이 포함된 1페이지 대시보드
- **L2**(심층 분석) — 피라미드 계층화와 교차 검증이 포함된 전체 분석 보고서

**S4 — QA**: `credit-qa-verifier` 스킬은 납품 전 품질 게이트 검토를 수행하여 신호 밀도 규칙, 원샷 거부 상한 준수, Mode B 가드레일, 단일 진실 공급원 무결성을 강제합니다. 이것이 4단계 체인의 종단 단계입니다 — QA를 통과하지 않으면 보고서가 납품되지 않습니다.

**실행 가능 오케스트레이터**: `src/pipeline.py`는 코드에서 4단계 체인 전체를 구동합니다. `dev/engine/pipeline-contract.md`에서 단계 정의를 읽고(단계 이름을 하드코딩하지 않음), `src/path_sheet.py`를 사용하여 경로 시트를 검증하며, 명시적으로 유선된 경로에 대해서만 코딩된 엔진을 호출합니다. 4개 산출물(경로 시트, 분석 산출물, 납품 시트, QA 판정)과 체인 엣지의 단일 진실 공급원은 `dev/engine/pipeline-contract.md`입니다.

## FAQ

### Q1: Credence는 전통적인 신용 분석과 어떻게 다른가요?

Credence는 전통적인 재무 분석의 세 가지 체계적 실패를 해결합니다: (1) 가장 중요한 신용 요소(정책, IP, 리스 품질)는 대차대조표에 절대 나타나지 않음, (2) 외부 등급은 실제 악화보다 평균 17개월 이상 지연, (3) 단일 관점 분석은 구조적 약점을 놓침. Credence는 계층화된 피라미드 프레임워크, 이중 트랙 교차 검증, 모자이크 데이터 조립, 다중 이해관계자 관점을 사용하여 전통적 분석이 놓친 것을 밝혀냅니다.

### Q2: 유료 구독이나 API 키가 필요한가요?

아니요. Credence는 무료 공개 데이터 소스(SEC EDGAR, FRED, 중앙은행 포털, TRACE, 산업 협회 보고서, 웹 검색)로 작동합니다. POC 단계는 의도적으로 데이터 소스를 제한하여 공개 데이터만으로 효과적인 신용 분석이 가능함을 증명합니다.

### Q3: 엔진은 어떤 신용평가기관과 연계되나요?

엔진의 내부 등급 척도는 S&P(AAA~D), Moody's(Aaa~C), Fitch(AAA~D)에 매핑됩니다. 전체 정렬 표는 `dev/engine/dual-track-methodology.md`를 참조하십시오.

### Q4: Credence는 모든 AI 에이전트 CLI와 사용할 수 있나요?

네. Credence는 CLI에 의존하지 않습니다. Claude Code(완전 자동 발견), Codex, Cursor, Gemini, OpenCode에서 작동합니다. 저장소 루트의 `AGENTS.md` 파일이 범용 진입점 역할을 합니다.

### Q5: 16개 작업 경로는 어떻게 구성되나요?

경로는 바이사이드 역할별로 구성됩니다: 신용 선택자(2), 포트폴리오 매니저(2), 리스크 오피서(4), 트레이더(1), 어드바이저(1), 개인 투자자(1), 메타/특수 목적(5). 각 경로는 엔진 시퀀스, 보고서 템플릿, 품질 게이트를 지정합니다. 8개 경로가 완전히 활성, 6개가 부분 구현, 2개가 계획됨입니다.

### Q6: 6개 국제 패러다임은 무엇인가요?

경기 순환형(P1), 방어형(P2), 성장(P3), 규제 유틸리티(P4), 금융(P5), 국채 연계(P6)입니다. 각 패러다임이 산업 분석의 가중치 템플릿과 점수 우선순위를 결정합니다.

### Q7: "시스템 인텔리전스 계층"이란 무엇인가요?

단일 발행자 분석을 넘어 시스템 패턴을 감지하는 최상위 집계 계층입니다: 산업 간 전염(19x19 매트릭스), 포트폴리오 집중도(5차원), 시스템 위험 지수(SRI)와 4단계 온도계. 이 모듈들은 "이 단일 발행자가 위험한가"가 아닌 "포트폴리오/시장 전체에서 무슨 일이 일어나고 있는가"에 답합니다.

### Q8: 테스트와 CI 설정은?

pytest 회귀 스위트는 코드화된 엔진, 파이프라인 오케스트레이터, 문서 레지스트리, 일관성 검사기, 릴리스 패키지 빌더를 커버합니다. CI는 Python 3.11 및 3.12에서 GitHub Actions를 통해 실행됩니다. 로컬 실행: `python -m pytest tests/ -q`. 문서 간 일관성은 `python scripts/consistency_check.py`로 별도 검증됩니다.

### Q9: 어떤 재무 보고 기준이 지원되나요?

IFRS(국제재무보고기준)와 US GAAP(일반적으로 인정된 회계 원칙)을 모두 지원합니다. 엔진은 발행자의 제출 서류에서 보고 기준을 자동 감지하고 주요 차이점을 조정합니다.

### Q10: 4개 스킬 체인을 어떻게 시작하나요?

CLI에서 패키지 루트를 열고 "[Y] 업계의 [X] 회사 분석해줘"라고 말하십시오. `credit-analysis-router` 스킬이 라우팅을 처리합니다. 직접 스킬을 사용하려면 `AGENTS.md` 지침에 따라 관련 `SKILL.md`를 로드하십시오.

### Q11: 코딩된 엔진과 LLM 오케스트레이션 경로의 차이점은 무엇인가요?

4개 작업 경로에 전용 Python 구현(코딩된 엔진)이 있습니다: 집중도 점수, 전염 매트릭스, SRI 계산, 전망 모니터링. 이들은 경로 ID가 일치하면 오케스트레이터(`src/pipeline.py`)가 직접 호출합니다. 다른 모든 경로는 엔진 문서를 참조하여 LLM이 오케스트레이션합니다 — Python 코드가 방법론을 복제하지 않습니다.

### Q12: Credence를 상업적 목적으로 사용할 수 있나요?

상업적 사용에는 저작권자의 사전 서면 허가가 필요합니다. 전체 조건은 [LICENSE](LICENSE) 파일을 참조하십시오. 엔진은 비상업적 평가, 연구, 내부 평가를 위해 소스 열람 가능 형태로 제공됩니다.
