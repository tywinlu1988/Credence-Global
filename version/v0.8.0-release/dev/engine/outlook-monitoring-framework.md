# 评级展望与持续监控框架

**版本**: v0.8.0-release | **日期**: 2026-07-10 | **定位**: 在双轨静态评级基础上，增加前瞻方向和持续监控机制

---

## 一、为什么需要展望与监控

### 1.1 当前引擎的缺口

当前引擎（v0.3.0）的输出结构为**静态快照评级**（AAA至D），仅反映分析时点的信用质量判断。专业评级机构在静态评级之外，始终维持**三个时间维度**：

| 时间维度 | 传统评级产出 | 引擎当前状态 | 缺口 |
|---|---|---|---|
| **当前时刻** | 评级（AAA-D） | 已实现 | — |
| **12-24个月** | 展望（正面/稳定/负面） | **缺失** | 无法传达信用质量的方向性变化趋势 |
| **90天内** | 观察名单（正面观察/负面观察） | **缺失** | 无法对近期的信用事件做出快速反应 |

### 1.2 缺失两项带来的实际问题

| 场景 | 没有展望 | 没有观察名单 |
|---|---|---|
| **评级维持BB+但趋势恶化** | 用户误以为"风险可控" | — |
| **突发资产划转公告** | — | 用户无法评估90天内是否需行动 |
| **行业政策急转弯** | 评级不变但风险已大幅上升 | 缺乏预警时间窗口 |
| **多个信号方向不同** | 无法表达"方向判断不一致"的不确定性 | — |

### 1.3 展望与观察名单在整体架构中的位置

```
引擎输出 (v0.3.0 之前)
  ├── 评级 (AAA ~ D)                         ← 静态快照
  └── 信号完备性报告

引擎输出 (v0.3.0 + 本模块)
  ├── 评级 (AAA ~ D)                         ← 当前信用质量判断
  ├── 展望: 正面 / 稳定 / 负面               ← 12-24个月方向性判断  ★新增
  ├── 观察名单: 正面观察 / 负面观察 / 无      ← 90天内可能评级行动  ★新增
  ├── 持续监控清单                            ← 主动监控的关注项    ★新增
  └── 信号完备性报告
```

## 二、评级展望机制（12-24个月方向性判断）

### 2.1 展望的定义与概率标准

展望是对未来12-24个月内评级变动方向的**概率性判断**，不是精确预测。

| 展望 | 定义 | 概率阈值 | 时间窗口 |
|---|---|---|---|
| **正面** | 未来12-24个月内评级上调1-2子级的可能性 ≥ 50% | ≥ 50% | 12-24个月 |
| **稳定** | 未来12-24个月内评级维持不变的概率 ≥ 70% | ≥ 70% | 12-24个月 |
| **负面** | 未来12-24个月内评级下调1-2子级的可能性 ≥ 50% | ≥ 50% | 12-24个月 |
| **发展中** | 未来12-24个月内评级可能上调或下调，方向尚不明确 | — | 12-24个月 |

**附加定义**：

| 概念 | 说明 |
|---|---|
| **展望调整窗口** | 展望评级自发布之日起12-24个月内有效；期满后需重新评估 |
| **子级变动** | 1子级 = 一个评级台阶（如BBB+到BBB-），2子级 = 两个台阶（如A-到BBB+） |
| **方向不确定时的处理** | 当正面和负面信号同时存在且权重相当，使用"发展中"展望，并标注双向触发条件 |

### 2.2 展望的触发因子矩阵

从引擎现有金字塔各层（L1-L4）及外部支持层中提取展望信号。每个触发因子附带**数据来源**和**信号方向**。

#### 正面展望触发信号

| 金字塔层 | 正面展望触发信号 | 数据来源 | 验证方式 |
|---|---|---|---|
| **L1 政策/宏观** | 行业政策明确支持力度加大（补贴延续/税收优惠扩大/产能指标放宽） | 工信部/发改委/财政部官网公告 | 政策文件原文+行业协会解读交叉验证 |
| **L1 政策/宏观** | 行业进入国家战略支持名单（如集成电路大基金/新能源专项） | 国务院/发改委产业规划文件 | WebSearch政策文本解析 |
| **L1 政策/宏观** | 金融监管导向趋向宽松（信贷额度增加/发债审批加速） | 央行/银保监会政策文件 | 公开市场操作公告+银行间市场数据 |
| **L2 技术/竞争** | 技术路线被确认为行业主流（如BC技术获央企集采>50%份额） | 央企集采结果公告、行业协会白皮书 | WebSearch招投标数据交叉验证 |
| **L2 技术/竞争** | 获关键市场准入资质（如创新药NDA批准/医疗器械注册证获批） | 国家药监局/CDE公告 | 政务公开平台公告确认 |
| **L2 技术/竞争** | 市场份额持续扩大（连续2+季度市占率提升） | 行业协会数据、券商研究报告 | WebSearch行业数据聚合 |
| **L3 供应链/运营** | 垂直一体化建成投产（原材料自给率提升至>50%） | 公司公告/环评公示/竣工公告 | 省级发改委项目备案查询 |
| **L3 供应链/运营** | 海外产能投产（规避贸易壁垒/汇率风险） | 公司公告、东道国政府公示 | 跨境项目备案+驻外经商处公告 |
| **L3 供应链/运营** | 核心客户签订长期供应协议（>3年锁定） | 重大合同公告（上证/深交所公告平台） | 交易所公告原文 |
| **L4 财务** | 经营现金流连续2+季度转正 | 季报/年报现金流量表 | 财报结构化数据提取 |
| **L4 财务** | FCF/收入持续改善（连续2+季度上升>2%） | 季报/年报 | 跨期FCF趋势计算 |
| **L4 财务** | 利息覆盖倍数连续2+季度回升（EBITDA/利息>3x） | 季报/年报 | 财务比率计算验证 |
| **L4 财务** | 短债/总负债比例持续下降（下降>5%/季度） | 季报/年报资产负债表 | 负债结构趋势分析 |
| **外部支持** | 省政府/国资委明确表态提供资金或资源支持 | 地方政府公告、国资委会议纪要 | WebSearch官方信息源 |
| **外部支持** | 集团/母公司计划注资或资产注入 | 公司公告、重大资产重组预案 | 交易所公告 |

| **L4 财务** | 利息覆盖倍数持续恶化（EBITDA/利息<1.5x且持续>2季度） | 季报/年报 | 财务比率恶化趋势确认 |

| **L4 财务** | 货币资金大幅减少（季度环比下降>30%无合理解释） | 季报/年报 | 跨期货币资金变动分析+审计意见核查 |
| **L4 财务** | 资产负债率超过行业阈值（政策驱动型>80%/技术壁垒型>70%） | 季报/年报 | 行业对标+负债率趋势 |
| **L4 财务** | 被出具非标审计意见（强调事项/保留意见/无法表示意见） | 审计报告 | 审计意见类型+强调事项具体内容 |
| **外部支持** | 核心资产被无偿划转（如华晨集团模式） | 公司公告、国资委国有资产划转披露 | 交易所公告 |
| **外部支持** | 支持方（省政府/集团）自身财务恶化 | 政府财政报告、集团财报 | 地方财政数据+集团报表交叉验证 |
| **外部支持** | 支持方信用评级被下调 | 评级机构公告 | WebSearch评级行动公告 |

### 2.3 展望评估的计分逻辑

无法精确计算"展望概率"——但可以使用信号方向的一致性强度来评估展望置信度。

**信号计数规则**：

```
正面信号计数   = 每检测到一个正面触发信号计数+1
负面信号计数   = 每检测到一个负面触发信号计数+1
信号方向权重 = 因子的金字塔层级权重（L1 > L2 > L3 > L4）

净方向判断信号 = Σ(正面信号 × 所在层权重) - Σ(负面信号 × 所在层权重)

如果净方向判断信号 ≥ +2.0 且 正面信号数 ≥ 2 → 正面展望
如果净方向判断信号 ≤ -2.0 且 负面信号数 ≥ 2 → 负面展望
如果 -1.9 < 净方向判断信号 < +1.9 → 稳定展望
如果 正面和负面信号均 ≥ 3 且方向信号强度接近 → 发展中展望
```

**置信度分级**：

| 置信度 | 条件 | 含义 |
|---|---|---|
| **高** | 4+个信号方向一致，覆盖≥3个金字塔层级 | 方向性判断很可靠 |
| **中高** | 3个信号方向一致，覆盖≥2个金字塔层级 | 方向性判断较可靠 |
| **中** | 2个信号方向一致，覆盖≥1个金字塔层级 | 方向性判断有参考价值 |
| **低** | 仅1个信号或所有信号来自同一层级 | 方向性判断仅作为提示 |
| **极低** | 仅有0-1个可识别的信号 | 数据不足以形成方向判断 |

### 2.4 展望的输出规范

```yaml

展望: 负面
展望有效期: 2026-07-08 至 2028-07-07
展望触发因子:
  - L2技术: TOPCon产能占比>90%，BC路线对其构成结构性替代威胁（来源: 2026Q2央企集采结果公告）
  - L4财务: 经营现金流连续4个季度为负，恶化趋势未逆转（来源: 2026Q1季报现金流量表）
  - 外部支持: 地方政府债务率>150%，支持能力边际减弱（来源: 2025年地方政府财政决算报告）
正面因子（如有）:
  - L3供应链: 海外产能已投产，2026H2预期贡献收入（来源: 公司2026年4月产能公告）
展望置信度: 中高（3/4个信号方向一致，覆盖3个层级）
负面情景: 若2026H2经营现金流仍未转正且地方支持未落实，评级下调1-2子级概率>60%
正面情景: 若海外产能贡献超预期且政府出台行业救助政策，展望可调至稳定（概率<20%）
```

```yaml
  - L4财务: FCF/收入从-2%改善至+3%，连续2个季度为正（来源: 2026Q1季报）
展望置信度: 中（3个信号但主要来自L1和L4，L2信号有待确认）
负面情景: 无显著负面因子
正面情景: 若L2市占率提升在2026Q3确认，评级上调1子级概率>50%
```

```yaml
评级: CCC
展望: 发展中
展望有效期: 2026-07-08 至 2028-07-07
展望触发因子:
  负面方向:
    - L4财务: 利息覆盖倍数0.8x，短债占比72%，随时可能触发违约（来源: 2026Q1季报）
    - L3供应链: 核心供应商已暂停供货（来源: 供应商公告/行业报道）
  正面方向:
    - 外部支持: 省政府已表态将协调纾困资金（来源: 省政府专题会议纪要，WebSearch确认）
    - L2技术: 核心专利已质押给银行作为新增贷款条件（来源: 国家知识产权局质押登记公告）
展望置信度: 中（正面和负面信号力量相当，发展走向取决于政府纾困是否落地）
双向触发条件:
  - 若省纾困资金在2026Q3内到位 → 展望可调至稳定，评级维持CCC或上调至B-
  - 若纾困资金未落地 → 展望调至负面，评级下调至D
```

### 2.5 展望与现有评级映射的衔接逻辑

展望不是独立判断——是基于现有评级映射结果的**方向延伸**。

```
        轨道A评分 + 轨道B评分 → 交叉对撞 → 综合评级 → 展望  ← 基于信号方向判断
                                                    ↓
                                              观察名单 ← 基于事件判断
```

**衔接规则**：

| 当前评级 | 负面展望 | 稳定展望 | 正面展望 |
|---|---|---|---|
| AAA | 极少见（维持率>95%） | 默认状态 | 不存在（AAA已为最高） |
| AA / A | 技术上可行 | 最常见 | 较少见 |
| BBB / BB | 最常见 | 最常见 | 较少见 |
| B | 常见 | 最常见 | 较少见 |
| CCC | 常见（通常伴随负面观察） | 少见 | 极少见（仅当有明确支持信号） |
| D | 不存在 | 不存在 | 不存在（D为终局状态） |

**关键约束**：
- **AAA评级不能有正面展望**（已为最高等级）
- **D评级不赋予展望**（终局状态不需要方向判断）
- **CCC评级与负面展望或负面观察应为默认配对**
- **展望方向与当前评级有偏离时需特别说明**（如"B+/正面展望"意味着远期内可能上调，但当前评级受限于短期风险因素）

## 三、观察名单机制（90天内可能的评级行动）

### 3.1 进入观察名单的触发条件

观察名单针对**近期事件驱动**的评级变化可能，分为"正面观察"和"负面观察"。与展望不同——观察名单是基于明确事件信号，不是趋势判断。

#### 负面观察触发条件

| 触发类型 | 具体条件 | 数据来源 | 信息时滞 |
|---|---|---|---|
| **事件驱动** | 重大资产重组公告（出售核心子公司/主要经营性资产） | 交易所公告 | 实时（T+1日内） |
| **事件驱动** | 核心子公司股权变更/被无偿划转 | 国资委公告、工商变更登记 | T+1~3日 |
| **事件驱动** | 实际控制人变更/被采取强制措施 | 公司公告、裁判文书网、执行信息公开网 | 实时 |
| **事件驱动** | 被监管立案调查（证监会/交易所立案） | 公司公告、证监会公告 | 实时 |
| **事件驱动** | 核心管理层失联/非正常离职（CEO/CFO） | 公司公告、媒体报道 | 实时 |
| **事件驱动** | 债券持有人会议触发+重大不利议案通过 | 债券公告 | T+1日 |
| **财务突变** | 突发大额亏损（单季度亏损超过去三年净利润之和的50%） | 季报/业绩预告 | T+1日 |

### 3.2 观察名单管理规则

| 管理项 | 规则 |
|---|---|
| **进入条件** | 触发任一进入条件 → 自动进入对应观察名单；同一主体可进入"负面观察"或"正面观察"（不可同时进入两方） |
| **观察期限** | 进入日起90天内自动完成重评；特殊情况可延长一次（最长60天），需在进入满60天前记录延期原因 |
| **重评时限** | 负面观察：30天内完成初步评估，60天内完成重评；正面观察：60天内完成重评 |
| **退出条件** | (1) 90天期满且事件明朗化，输出正式重评；(2) 触发事件在期限内被证明为误读或已逆转；(3) 违约/评级行动已实际发生——观察名单使命完成 |
| **清单公示** | 观察名单主体应形成定期更新清单，标注进入日期、触发事件、当前状态 |

### 3.4 展望 vs 观察名单的边界和协作

两个机制容易混淆，需要明确边界：

| 维度 | 展望 | 观察名单 |
|---|---|---|
| **时间窗口** | 12-24个月 | 90天 |
| **驱动因素** | 趋势性信号（政策方向/技术路线/财务趋势） | 事件性触发（公告/突变/突发政策） |
| **信号类型** | 多信号累加/方向一致性 | 单事件触发 |
| **输出频率** | 随评级周期（季度/年度） | 实时（事件触发即评估） |
| **概率含义** | ≥50%概率评级变动 | 高度可能近期评级变动 |
| **评级变化幅度** | 1-2子级 | 通常>1子级 |

**协作规则**：

```
同一主体的展望和观察名单可以同时存在：

情景1: 负面展望 + 负面观察
  → 信用质量本来趋势较差 + 近期发生了触发事件
  → 默认动作：观察期内启动重评，预期下调评级

情景2: 稳定展望 + 负面观察
  → 本来趋势正常 + 突发负面事件
  → 默认动作：快速评估事件影响，若确认重大负面影响→调整展望至负面并下调节点

情景3: 负面展望 + 无观察名单
  → 趋势恶化但尚未触发具体事件
  → 默认动作：维持展望，按季度重检

情景4: 正面展望 + 负面观察（矛盾情景）
  → 长期趋势正面但短期遭遇突发负面事件
  → 默认动作：以观察名单为重（短期事件优先），若负面影响可控则维持正面展望
```

## 四、持续监控的触发机制

### 4.1 设计原则

监控不是被动等待用户查询——是引擎主动对关注列表和持仓进行持续扫描。以下框架定义了监控**什么**、**何时**监控、以及什么条件下**触发重评**。

### 4.2 监控矩阵

| 监控项 | 频率 | 数据来源 | 触发重评条件 | 优先级 |
|---|---|---|---|---|
| **政策变化** | 实时（新政策发布24小时内） | 国务院/发改委/工信部/央行官网 | 政策影响范围覆盖关注企业所属行业 | P0 |
| **财务数据** | 季报/年报发布后48小时内 | 交易所公告、中国货币网 | 任一L4指标恶化超过阈值（短债占比>60%/利息覆盖<1.5x/FCF连续2季为负） | P0 |
| **信用事件** | 实时 | 中国货币网、交易所公告、评级机构公告 | 评级调整/违约/展期/破产申请/被申请强制执行/失信被执行人 | P0 |
| **市场信号** | 日度（交易时间） | 交易所债券数据、中债估值、上清所 | 利差单日跳变>50bp / 债券价格单日波动>10% / 北向资金单周急剧流出>20% | P1 |
| **管理层变动** | 实时 | 公司公告、企查查/天眼查工商变更 | CEO/CFO/实际控制人变更/被采取强制措施/失联 | P0 |
| **资产交易** | 实时 | 交易所重大资产重组公告、国资委公告 | 重大并购（交易金额>净资产20%）/ 出售核心资产 / 无偿划转 | P0 |
| **司法执行** | 日度 | 中国执行信息公开网、裁判文书网、企查查司法数据 | 新增被执行记录/股权冻结金额>净资产10% | P0 |
| **股权质押** | 周度 | 中登公司质押数据 | 大股东质押比例>80% / 触及平仓线 / 补充质押频率异常 | P1 |
| **审计意见** | 年报披露季后30天内 | 年度审计报告 | 审计意见变化（标准→非标/非标→更严重非标） | P0 |
| **行业竞争** | 月度 | PVInfoLink/TrendForce/行业协会数据 | 核心产品价格跌破全行业成本线 / 行业库存>3个月 | P1 |
| **资金流向** | 周度 | 交易所资金流向数据、公募基金季报 | 行业配置比例连续2季度下降 / 重仓基金数量大幅减少 | P2 |
| **舆情监控** | 日度 | WebSearch媒体监测 | 涉及财务造假/重大诉讼/产品重大质量问题的负面报道 | P1 |

### 4.3 推送优先级的智能排序

监控信号不全部推送给用户。以下三维度排序决定推送的**紧迫性**和**呈现方式**。

**维度定义**：

| 维度 | 分级 | 评分 |
|---|---|---|
| **紧急性**：事件在90天内导致评级变化的概率 | 紧急（≥60%概率） | 3分 |
|  | 较急（30-60%概率） | 2分 |
|  | 一般（<30%概率） | 1分 |
| **重要性**：事件的潜在评级影响幅度 | >3子级 | 3分 |
|  | 1-2子级 | 2分 |
|  | <1子级 | 1分 |
| **稀缺性**：用户是否已知（重复信号降权） | 首次出现的信号 | 3分 |
|  | 重复但未充分确认的信号 | 2分 |
|  | 已多次推送过的信号 | 1分 |

**推送优先级计算**：

```
推送优先级分数 = 紧急性 × 重要性 × 稀缺性

27分（3×3×3）→ 最高优先级：立即Push推送，标红告警
18-24分        → 高优先级：当日报送中置顶呈现
9-16分         → 中优先级：列入周度监控报告
3-8分          → 低优先级：纳入月度趋势汇总
1-2分          → 归档级别：存入历史记录，无需主动推送
```

**推送渠道建议**：

| 优先级 | 推送方式 | 频率 |
|---|---|---|
| 最高（27分） | 实时Push告警 + 邮件通知 | 立即 |
| 高（18-24分） | 日度简报置顶 | 每工作日 |
| 中（9-16分） | 周度监控报告 | 每周 |
| 低（3-8分） | 月度趋势汇总 | 每月 |
| 归档（1-2分） | 不需要主动推送 | 可查询 |

### 4.4 监控信号的历史记录管理

每条监控信号应建立结构化的历史记录，用于趋势判断和信号密度计算。

```yaml
监控记录ID: MON-2026-07-08-001
主体: [企业名称]
监控项: 信用事件 - 评级调整
信号摘要: 中诚信国际将主体评级从AA+/负面下调至AA-/负面
来源: 中诚信国际公告 2026-07-08
优先级评分: 27（紧急3×重要3×稀缺3）
推送状态: 已推送（2026-07-08 09:30）
用户响应: 待确认
关联重评: 待触发
```

**信号密度计算**：每条监控记录计入对应金字塔层的信号密度分子。

```
某层信号密度 = 该层已有信号数 / 该层目标信号数（按行业类型预设）

当信号密度 > 阈值（政策驱动型 L1阈值70%；技术壁垒型 L2阈值70%）：
  → 触发综合重评推荐
```

## 五、评级迁移矩阵

### 5.1 中国信用债市场迁移概率参考

基于市场历史数据的近似估计。**重要限制见数据质量说明**。

#### 1年期迁移概率矩阵

| 当前评级 | 上调概率 | 维持概率 | 下调概率 | 违约概率 | 说明 |
|---|---|---|---|---|---|
| **AAA** | 0% | 95%+ | <5% | <0.5% | AAA已为最高，上调概率为0；违约率极低但非零（北大方正、华晨均曾为AAA） |
| **AA+** | 3-5% | 88-92% | 5-8% | <1% | 维持率较高，上调概率有限 |
| **AA** | 5% | 85% | 10% | 1-2% | 开始出现明显的下调风险 |
| **AA-** | 5-8% | 80-85% | 10-15% | 2-3% | 上下波动开始增大 |
| **A+** | 5-8% | 78-82% | 12-18% | 3-5% | 下调概率开始超过上调 |
| **A / A-** | 5% | 75% | 15-20% | 5% | 信用质量分化明显 |
| **BBB+** | 5-8% | 70-75% | 18-25% | 5-8% | 进入投机级边缘 |
| **BBB / BBB-** | 3-5% | 65-70% | 20-30% | 8-10% | 投机级特征显现 |
| **BB** | 3-5% | 60% | 25-30% | 10-15% | 高风险区 |
| **B** | <3% | 50-55% | 30-35% | 15-20% | 半数主体在1年内被下调或违约 |
| **CCC** | <2% | 30-40% | 30-40% | 30%+ | 三年内超半数违约 |
| **D** | 0% | — | — | 100% | 终局状态 |

### 5.2 数据质量与使用限制

| 限制项 | 具体说明 |
|---|---|
| **样本量有限** | 中国信用债市场的历史仅约20年（2005年正式启动），实质性违约案例集中在2018年后，样本量远小于美国市场（超过100年数据） |
| **评级虚高问题** | 外部评级中枢严重偏高（AAA和AA级占存量信用债的80%以上），现阶段的迁移概率可能低估实际违约风险 |
| **尾部风险缺失** | 缺少跨完整经济周期的违约样本——中国市场尚未经历一次完整的"繁荣→衰退→复苏"信用周期 |
| **非随机性** | 中国信用债违约存在明显的行业集聚性和"国企信仰"扭曲效应，迁移概率不能简单外推 |
| **结构性变化** | 城投/地产/产业债的违约特征各不相同，单一迁移矩阵无法反映结构性差异 |

**诚实声明**：
> 上述迁移矩阵是基于有限历史数据的**方向参考**而非精确概率预测。矩阵的统计显著性不足以支撑量化违约概率计算。在引擎输出中，迁移矩阵仅用于：(1) 作为展望判断的粗校准参考；(2) 在持续监控中标记概览性的评级变化概率范围。不建议将迁移概率直接用于定价或风险计量。

### 5.3 行业差异化的迁移概率修正

不同行业因其结构性特征，迁移概率应符合行业特定的方向偏移：

| 行业类型 | 相对全市场的迁移偏差 | 原因 |
|---|---|---|
| **政策驱动型** | 下调概率+5-10%（政策驱动型） | 政策急转弯可快速摧毁行业信用质量；L1层权重>35%→政策一票否决风险 |
| **技术壁垒型** | 上调概率+3-5% | 核心IP/注册证构成较强护城河，一旦建立优势可快速提升 |
| **存量博弈型** | 下调概率+5% | 零和博弈中\>30%企业会被淘汰，尾部风险更高 |
| **资产租约型** | 上调概率+2% | 类REITs属性，商业模式稳定性较高 |
| **半导体（特殊）** | 下调概率+10-15% | 地缘政治风险是最重因子，五层金字塔中L1占30-35%，调整幅度可能更大 |
| **新能源车-OEM** | 下调概率+5-10% | 竞争激烈+技术迭代快+L5政策变动风险 |

## 七、公开数据约束下的展望可靠性

### 7.1 本框架展望的固有局限性

| 局限性 | 原因 | 对使用方式的约束 |
|---|---|---|
| **展望是方向性判断，不是精确预测** | 信号计分逻辑基于可获取公开数据的数量和一致性，不是量化违约概率模型 | 不能将展望等同于"评级变化概率"，只能作为方向参考 |
| **统计基础有限** | 中国信用债市场缺乏跨完整周期的违约样本（见5.2节） | 迁移矩阵仅作为粗校准，不可用作VaR/PD的输入 |
| **时效性依赖数据更新** | 政策文件/财报/市场数据的发布存在天然滞后期 | 持续性监控最多缩短信息落后窗口，无法消除 |
| **公开数据信息含量不足** | 非上市企业无市场定价和实时财务数据；非标资产穿透数据不可得 | 对非上市主体的展望置信度自然低于上市主体 |
| **不可抗力不在预测范围** | 政策突然转向、金融危机、战争、疫情等系统性外部冲击 | 展望不覆盖这些事件的触发概率 |
| **集团风险穿透困难** | 合并报表掩盖了母公司单体风险，核心资产可能在子公司层级 | 对集团控股型主体的展望需要特别标注"集团穿透程度" |

### 7.2 不同数据可获取性下的展望置信度

| 主体类型 | 数据可得性 | 展望置信度 | 备注 |
|---|---|---|---|
| **上市企业（有债券存量）** | 高（财报+审计+市场定价+评级+公告） | 中高 | 最优条件 |
| **上市企业（无债券存量）** | 中高（有市场定价但无债券利差） | 中 | 缺少信用利差信号 |
| **非上市企业（有发债/披露要求）** | 中（有年报+债券公告但无市场定价） | 中 | 缺少市场信号，依赖基本面 |
| **非上市企业（无发债/无披露义务）** | 低（仅有司法/招标/招聘等碎片化信息） | 低 | 展望仅有方向性提示作用 |
| **单一项目主体（SPV）** | 中（有项目财报+付息记录但结构复杂） | 中 | 资产端数据优于主体端 |

### 7.3 展望陈述的标准免责声明

所有包含展望的引擎输出必须附带以下声明（或等效表述）：

> **展望声明**：本框架对[企业名称]给出的展望（正面/稳定/负面/发展中）是基于公开可获取数据的**方向性判断**，不代表量化评级变化概率，不构成对未来信用事件的精确预测。展望的有效期为12-24个月，期满或发生重大触发事件时应重新评估。政策突然转向、金融危机、不可抗力等系统性外部冲击不在本展望的预测范围内。本框架不对因依赖本展望而产生的任何投资决策或财务损失承担责任。
