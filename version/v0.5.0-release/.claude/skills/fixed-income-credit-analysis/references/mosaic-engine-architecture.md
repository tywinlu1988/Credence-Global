# 马赛克分析引擎 — 系统架构设计

**版本**: v0.1
**日期**: 2026-07-08
**核心原则**: 承认数据永远残缺，把"残缺"从弱点变成产品特征

---

## 一、核心理念

### 1.1 两个基础理论

| 理论 | 含义 | 在引擎中的体现 |
|---|---|---|
| **马赛克理论** | 单块碎片无意义，拼在一起构成完整画像。每一片公开信息——政策文件、法院公告、招标结果、价格周报——单独看都不能说明问题，但聚合后形成判断 | 多源数据聚合 → 信号叠加 → 置信度加权 |
| **信息完备性理论** | 不仅回答"知道了什么"，更回答"还不知道什么"。信息缺口不是缺陷，而是风险信号的来源——"我们没有这个数据"本身就是在告诉用户"这个维度存在不确定性" | 每个分析结论附带"数据完备性评分"和"缺口列表" |

### 1.2 马赛克拼图的五层信号

```
                 ┌──────────────────────────────┐
  高置信度       │  第五层：多源交叉验证信号       │  同一事实被≥2个独立来源确认
      ↑          │  第四层：单源直接信号           │  一个可靠来源明确陈述
      │          │  第三层：衍生推断信号           │  从多片碎片组合推理
      │          │  第二层：方向性弱信号           │  只有方向，没有量级
  低置信度       │  第一层：缺失信号（缺口）        │  应该有的数据没有
                 └──────────────────────────────┘
```

**每一片数据都标注它属于哪一层。** 五层交叉验证的结论 > 单源直接信号 > 推理 > 弱信号。缺口单独标注。

## 三、核心模块详细设计

### 3.1 信号提取层

**输入**：WebSearch返回的搜索结果（标题+摘要+URL）
**输出**：结构化信号对象

每个信号的结构：

```yaml
signal:
  id: "sig-001"
  source_url: "https://..."
  source_type: "policy_document" | "court_record" | "bidding_result" | "industry_report" | "news"
  timestamp: "2026-06-15"  # 信息来源的时间，不是抓取时间
  dimension: "L2_technology"  # 属于金字塔哪一层
  content: "隆基BC电池量产效率达24.8%"
  direction: "positive" | "negative" | "neutral"
  strength: 1-5  # 1=弱方向性信号, 5=明确量化信号
  cross_validated: false  # 初始都是false，进入拼图层后如有≥2独立源确认为true
```

**LLM的任务**：从每条搜索结果的标题+摘要中提取信号，不做判断，只做结构化。最多返回3条信号/源。

### 3.2 马赛克拼图层

**输入**：信号列表
**输出**：按金字塔维度聚合后的评估矩阵

**核心逻辑**：

```
对于每个分析维度（如L1政策环境）：
  1. 汇总该维度的所有信号
  2. 同方向信号叠加 → 增加置信度
  3. 矛盾信号出现 → 标注"分歧"并降低置信度
  4. 跨源交叉验证 → 提升信号层级
  5. 计算该维度的"信号密度" = 有效信号数 / 该维度应有信号数的预估
```

**信号密度是最关键的输出**——它直接回答"我们对这个维度了解多少"：

| 信号密度 | 含义 | 对最终评估的影响 |
|---|---|---|
| >80% | 信息充分——关键信号基本齐全 | 高置信度，可直接给出评分 |
| 50-80% | 信息中等——主要信号有但细节不足 | 中置信度，评分附带±1的误差区间 |
| 20-50% | 信息不足——只有碎片 | 低置信度，标注为"待补充"，给出方向性判断而非精确评分 |
| <20% | 信息严重缺失 | 不给出该维度评分，标注为"信息不足无法评估" |

### 3.3 完备性评估层（核心新增模块）

**这是产品差异化的关键。** 不仅告诉用户"我们看到了什么"，更告诉用户"我们没看到什么，这意味着什么风险"。

**对每个分析维度，输出三件事：**

```
维度：L2 技术路线与竞争位势
├── 信号密度：75%（信息中等）
├── 已获取的关键信号：
│   ├── ✅ 组件量产效率（隆基BC 24.8% vs TOPCon ~23.5%）
│   ├── ✅ 央企集采入围情况（2026年集采59GW数据）
│   ├── ✅ 专利数量（隆基510件 vs 晶科/天合）
│   └── ⚠️ 产能利用率（行业估算值，非企业披露）
├── 缺失的信号：
│   ├── ❌ 各企业的良率数据（企业不单独披露）
│   ├── ❌ 各技术路线的非硅成本精确对比（仅公开估算）
│   └── ❌ BC vs TOPCon在分布式场景的实际发电量对比（无第三方测试）
└── 缺口风险评估：
    → 良率数据缺失 → 无法精确判断成本竞争力 → 评分区间 ±1.5
    → 但央企集采入围数据和效率差距已构成足够的区分信号
```

**缺口→风险映射表（通用模板，按行业定制）：**

| 缺口类型 | 典型缺失数据 | 对应的信息风险 | 替代信号 |
|---|---|---|---|
| 竞争数据缺口 | 各企业成本精确对比 | 高估或低估成本优势 | 央企集采中标价（反映市场接受的溢价） |
| 财务数据缺口 | 非上市企业无公开财报 | 完全无法评估财务健康 | 被执行记录、司法纠纷、招聘动态 |
| 市场定价缺口 | 非上市企业无股价/债券 | 无法进行轨道B分析 | 公开融资事件、股权转让价格 |
| 条款数据缺口 | 无Wind终端无法获取久期/Z-spread | 无法做精细利率风险分析 | YTM近似、同评级利差对比 |
| 流动性数据缺口 | Bid-ask spread不可得 | 无法评估真实交易成本 | 日均成交量、换手率、异常成交 |

### 3.4 综合输出层

**输出的完整结构：**


## 六、一个实例：马赛克理论如何工作

以"判断通威股份的偿债能力"为例：

**五片碎片——每片单独看都不够：**

| 碎片 | 信号 | 单独看 |
|---|---|---|
| 通威短融2025年利率从2.60%→2.02%（-58bp） | 市场认为风险在降低 | 可能只是货币环境宽松 |
| 通威短融2026年1月利率从2.02%反弹至2.15%（+13bp） | 市场开始重新定价风险 | 反弹幅度太小，可能只是噪声 |
| 通威2025年亏损90-100亿 | 基本面严重恶化 | 亏损可能是一次性的（硅料价格暴跌） |
| 通威应收账款从Q3的XX亿跳升至Q4的YY亿 | 回款能力在恶化 | 可能只是季节性因素 |
| AAA/稳定评级维持 | 评级机构认为风险可控 | 评级滞后是已知问题 |

**拼在一起后：**

```
碎片1（利差压缩） + 碎片2（利差反弹） = 利差拐点信号（弱→中等强度）
碎片2（利差反弹） + 碎片3（巨额亏损） = 市场开始反映基本面（强度升级）
碎片3（亏损） + 碎片4（应收跳升） = 不仅亏损，回款也在恶化（双杀信号）
碎片5（AAA维持） + 碎片1-4 = 评级滞后确认（评级信号不可信）

马赛克拼图结论：通威的偿债能力在恶化，市场定价（利差反弹）开始反映这一趋势，
但AAA评级仍在滞后。信息完备性：中（缺口=没有通威母公司的独立财务数据）。
```
