# 统计审计报告：固收信用分析引擎定量分析方法论

**审计日期**: 2026-07-08 | **审计目标**: v0.3.0 定量分析方法论的统计严谨性 | **审计方**: 数据统计分析专家

---

## 第一部分：方法论的统计严谨性评估

### 1.1 总体评估矩阵

| 量化模块 | 统计严谨性 | 一句话评估 |
|---------|-----------|-----------|
| **利差指标定义与替代体系** | 中高 | 替代方案标注充分，但YTM利差近似Z-spread的误差范围(+-15-30bp)未给出统计推导依据 |
| **利差异常检测规则** | 中 | 阈值设定(50bp, Mean+-3sigma)缺乏行业异质性考量且未控制多重假设检验 |
| **波动率历史计算** | 中 | 多窗口并行是良好实践，但年化因子sqrt(252)假设收益独立同分布，未考虑波动率聚集性 |
| **波动率分解(单因子回归)** | 中低 | 24周窗口24个观测值做单因子回归勉强可用，但系统性/特质性分解偏误未被充分量化 |
| **波动率突变检测** | 低 | 波动率变化幅度分级(10-30%/30-50%/>50%)阈值缺乏统计依据，HV排序条件过于严格 |
| **多因子回归模型** | 中低 | 24观测值对3-4因子OLS回归的统计效力极低，诊断虽覆盖DW/VIF但未检平稳性与异方差性 |
| **因子时变检测** | 中低 | 滚动窗口内的beta变化检测存在自相关偏误，Chow检验/Bai-Perron结构断点检验未引入 |
| **Pearson相关性矩阵** | 中 | 24周窗口Pearson系数未报告置信区间，未检验相关性显著性(p值缺失)，未考虑非平稳序列的伪相关 |
| **尾部相关性** | 低 | 24周窗口5%分位数的尾部相关性估计仅基于约1-2个观测值，统计效力极低 |
| **相关性突变检测** | 中 | delta>0.3阈值缺乏抽样分布依据，Fisher z变换未用于相关性差异的统计显著性检验 |
| **单因子敏感性(久期近似)** | 中高 | 线性近似忽略凸性且已标注局限，统计处理得当 |
| **多因子情景加权** | 低 | 情景概率(20%/10%/3%/1%)赋值无实证校准，独立假设下的误差范围(10-20%)为专家估计 |
| **尾部风险情景构建** | 中 | 方法论合理但缺乏极值理论(Extreme Value Theory)的统计支撑 |
| **市场隐含信号提取** | 中 | 规则定义清晰但多数领先窗口估计(2-12周等)缺乏系统的Granger因果检验验证 |
| **数据质量标注体系** | **高** | 全文最扎实的部分——每一指标均有质量等级/局限/替代方案/误差范围，是统计透明度的范例 |

### 1.2 各模块评分分布

```
高         ██ 数据质量标注体系
中高       ██ 利差指标定义、单因子敏感性、尾部风险情景

中低       ███ 波动率分解、多因子回归、因子时变检测、尾部相关性
低         ██ 波动率突变阈值、多因子情景概率赋值
───────────────────────────────────────────────
```

## 第二部分：需修正的统计问题

### 2.1 利差分析模块

| # | 当前做法 | 问题所在 | 修正建议 | 优先级 |
|---|---------|---------|---------|-------|
| S-1 | 异常检测统一阈值：单日>50bp、20日累计>50bp | **忽略行业异质性**——高收益债行业的利差标准差远大于投资级行业。50bp对AAA级可能为3sigma事件，对BB级可能仅为1.5sigma事件。不同行业的利差波动率差异可达3-5倍。 | 改用**行业标准化的Z-score阈值**。为每个行业/评级组合计算各自的利差变化均值和标准差，异常阈值设为行业特定的Mean+-k*sigma。对样本不足的行业(观测值<60)，用贝叶斯收缩(Bayesian shrinkage)向全局均值靠拢。 | **高** |
| S-2 | 检测条件3：利差分位数>90%持续15日 | **阈值选择的统计依据不足且存在条件依赖**——90%分位数的定义依赖于历史窗口长度，连续15日的阈值设定缺乏多重假设检验校正。如果同时监控100只债券，每只债券有5个检测条件，总检测次数=500，纯粹随机情况下约产生25个假阳性信号(5%显著性水平)。 | 引入**Bonferroni或Benjamini-Hochberg校正**控制假发现率(FDR)。或改用"警戒线+确认线"双层设计：较宽松阈值触发标记，第二层更严格的统计检验确认。 | **高** |
| S-3 | 60交易日滚动窗口计算Mean+3sigma | **3sigma阈值假设利差变化服从正态分布**——但信用利差变化具有典型的厚尾(fat tail)特征(峰度>3)。在厚尾分布下，3sigma对应的真实概率远高于正态分布的0.27%。峰值时刻的实际假阳性率可能达到1-3%。 | 使用**非参数分位数法**：用利差变化的经验分布的99%分位数替代Mean+3sigma。或拟合t分布/广义极值分布(GEV)来更准确地估计尾部阈值。至少应标注"假设正态分布可能低估假阳性率"。 | **高** |
| S-4 | Z-spread替代误差标注为"+-15-30bp" | **误差范围缺乏统计支撑**——这似乎是基于专家判断而非系统性的经验校准。不同期限结构形状、不同评级水平下，YTM利差与Z-spread的偏差方向和大小可能系统性地不同。 | 建议：1) 如果有少量Wind/Bloomberg接入时段，做YTM利差 vs Z-spread的配对校准(至少20对样本); 2) 在校准完成前，将误差标注改为"未经验证的估算，误差方向已知但量级未经统计校准"。 | **中** |

### 2.2 波动率分析模块

| # | 当前做法 | 问题所在 | 修正建议 | 优先级 |
|---|---------|---------|---------|-------|
| S-5 | HV = sigma(r) * sqrt(252) | **年化因子sqrt(252)假设日收益率独立同分布(i.i.d.)**。但金融时间序列已知存在波动率聚集性(volatility clustering)和自相关。当收益率序列存在正自相关时，sqrt(252)会低估年化波动率；存在负自相关时会高估。 | 使用**Newey-West调整**的年化因子：sqrt(N / (1 + 2*sum(rho_k)))，其中rho_k为收益率自相关系数。或在极端情况下，改用已实现波动率(Realized Volatility)框架直接加总日度平方收益率。作为一个快速修复，至少标注sqrt(252)假设的偏差方向。(需实证检验不同债券类型的自相关结构) | **中** |
| S-6 | 等权重历史波动率(20/60/120日) | **等权重窗口对最近的观测值和最旧的观测值赋予相同权重**。当波动率发生结构性变化时，等权重HV的反应速度较慢；如果波动率近期升高，等权重HV仍会被远距离的低波动率观测值拉低。 | 增加**指数加权移动平均(EWMA)波动率**作为补充(lambda=0.94为RiskMetrics标准参数)。EWMA对近期观测值赋予更高权重，能更快反映波动率突变。HV_20d与EWMA的背离本身就是一个信号——若HV_20d << EWMA，说明近期波动率急剧上升。同时输出两者。 | **中** |
| S-7 | 波动率分解的单因子滚动回归(120日窗口) | **三个统计问题**：1) 120日窗口内假设beta恒定——在波动率市中失效；2) 仅用单一市场因子(沪深300)——行业特质因子未被分离；3) 系统性/特质性分解的方差比为点估计，无置信区间。 | 1) 引入**滚动beta的时变性诊断**：计算120日窗口内beta的滚动标准差(子窗口60日)，若子窗口beta变异系数>20%，应重新估计或切换到时变参数模型；2) 增加行业因子作为第二因子(如果数据允许)；3) 使用**Delta方法或Bootstrap**计算方差分解比的置信区间。 | **高** |
| S-8 | "HV_20d > HV_60d > HV_120d 持续5日"作为波动率持续性上升信号 | **条件过于严格导致信号稀疏**。三条HV曲线同时严格单调排列(短期>中期>长期)且持续5日的概率极低，尤其在高波动环境下短期内三条曲线可能交错。这可能导致大量真实预警被漏报(Type II error过高)。 | 放宽为一组**软条件**(满足其中两项即可)：1) HV_20d > HV_60d；2) HV_60d > HV_120d；3) HV_20d的5日移动平均在连续上升；4) HV_20d/HV_60d比率 > 1.05。同时标注触发条件的组合方式。 | **高** |
| S-9 | 波动率变化幅度分级：10-30%/30-50%/>50% | **分级阈值缺乏统计基础**。为什么10-30%是"关注"而30-50%是"异常"？这些阈值的Type I/Type II错误率未经验证。且"20%以上下降为无变化/降级"这个不对称处理(上升分三级，下降只一级)缺乏解释。 | 改用**基于历史分位数的分级**：>75%分位数=关注，>90%分位数=异常，>97.5%分位数=危机。下降侧用对称处理(<-75%分位数可视为正面信号)。或者，保留当前分级但标注"阈值需通过历史回测验证"。 | **中** |

### 2.3 多因子回归模块

| # | 当前做法 | 问题所在 | 修正建议 | 优先级 |
|---|---------|---------|---------|-------|
| S-10 | 滚动OLS回归：窗口24周(周度数据=24观测值)，因子3-4个 | **小样本下的严重统计效力不足**。24个观测值对4个回归因子(含截距)的自由度=19。当因子间存在相关性(VIF约3), R²=0.5时的统计检验力(power)极低——真实beta=0.5的因子只有约40%概率被检测为显著。beta估计值本身的标准误也很大。 | 1) **最小观测值从12提高到30**(即至少30周数据)，低于此标注"不可靠"；2) 在观测值不足时使用**贝叶斯回归**：对beta设置informative prior(引用行业历史beta分布)；3) 输出beta的90%置信区间而非点估计；4) 用**Lasso回归**做因子筛选(自动处理相关因子并做正则化)。 | **高** |
| S-11 | 未检验时间序列的平稳性 | **非平稳时间序列的回归会产生伪回归问题(spurious regression)**。信用利差往往是非平稳的(I(1)过程)，如果用非平稳的利差水平值对非平稳的因子做回归，即使两者完全无关，t统计量也可能高度显著。 | 在回归前必须执行：1) **ADF检验(Augmented Dickey-Fuller)**或KPSS检验判断利差序列和因子序列的平稳性；2) 如果非平稳，使用**一阶差分**(Delta Spread)代替利差水平值；3) 或执行**Engle-Granger协整检验**确认变量间存在长期均衡关系后，使用水平值回归。 | **高** |
| S-12 | 未检验异方差性 | **OLS回归假设残差同方差**。但在信用利差的时间序列中，波动率随市场条件变化(低波动/高波动时期交替)，残差方差通常是非恒定的(heteroskedastic)，这会导致beta估计的标准误有偏，进而影响统计推断。 | 增加：1) **Breusch-Pagan检验**或White检验诊断异方差性；2) 如果存在异方差性，使用**异方差稳健标准误**(HC/HAC standard errors, 如Newey-West估计)；3) 在输出模板中增加"异方差检验"一行。 | **高** |
| S-13 | 残差正态性检验p>0.05通过即视为ok | **正态性检验在24样本下的检验效力极低**。Shapiro-Wilk检验在n<30时对偏离正态的检测能力有限——真实非正态的残差分布(如有偏/厚尾)只有约30-50%概率被p<0.05检测到。 | 1) 不依赖正态性检验的p值，同时输出**残差的偏度(skewness)和峰度(kurtosis)**指标；2) 增加**Q-Q图**的定性描述(或自动化代码检测)；3) 如果残差明显非正态，使用**Bootstrap**估计beta的标准误(Bootstrap不依赖正态假设)。 | **中** |
| S-14 | R²从>0.6跌至<0.3视为"定价逻辑结构性变化" | **R²下降的可能原因很多**，不一定是定价逻辑变化。R²减少也可能是：1) 因子本身的方差缩小(宏观环境稳定时因子解释力天然下降)；2) 样本内的极端值(影响R²的杠杆点)；3) 噪声增大(市场流动性下降)。 | 区分R²下降的原因：1) 检查因子的方差是否变化(比较两段窗口内因子的标准差)；2) 同时检查残差标准差——如果残差标准差不变而R²下降，是因子方差缩小；如果残差标准差加大，才是"定价逻辑变化"；3) 增加**Chow检验**(结构断点检验)作为更严格的确认。 | **中** |

### 2.4 相关性分析模块

| # | 当前做法 | 问题所在 | 修正建议 | 优先级 |
|---|---------|---------|---------|-------|
| S-15 | 24周窗口的Pearson相关系数(周度数据) | **24个观测值估计的相关系数置信区间非常宽**。例如rho=0.50的Pearson系数(n=24)，95%置信区间约为(0.12, 0.75)——下限在"弱相关"范围而上限在"强相关"范围。直接使用点估计做判断(0.35 vs 0.72)会夸大差异的显著性。 | 1) 每次输出相关系数时附上**95%置信区间**(使用Fisher z变换)和**p值**(H0: rho=0)；2) 在相关性矩阵中用星号标注显著性水平(*p<0.05, **p<0.01, ***p<0.001)；3) "相关性从0.35到0.72"应报告Fisher z检验的显著性：z(r1)-z(r2)的标准化差值。 | **高** |
| S-16 | 尾部相关性：低于5%分位数时的同跌比例 | **在24周数据中，5%分位数对应约1-2个观测值**。基于1-2个数据点估计尾部相关性，估计量的方差极大——实际上是一个几乎无统计意义的指标。尾部相关性的标准误与有效样本量成反比，当尾部样本N_tail=1-2时，标准误>0.3。 | 有三种改进路径(按可行性排序)：1) **扩展窗口至52周及以上**，使尾部样本达到N>=5；2) **使用半参数方法**：用所有数据拟合一个Copula模型(如t-Copula)，然后从拟合的Copula中读取尾部相关系数(lambda_u/lambda_l)——这样用了全部数据的信息来推断尾部行为；3) 如果样本不足，**标注"尾部相关性估计不可靠，N_tail=<X"**，定位为方向性参考而非量化指标。 | **高** |
| S-17 | 相关性突变检测阈值：delta>0.3(单周) 或 mean_rho周度变化>0.1持续4周 | **阈值选择缺乏统计基础且互为矛盾**——0.3的周跳跃在Pearson系数量纲上非常大。从rho=0.3跳到rho=0.6(n=24)对应的Fisher z统计量约为1.14(p值约0.25)，并不统计显著。阈值被设得太高以至于达到时已经错过了早期预警时间。 | 改用**基于Fisher z变换的统计显著性检测**：1) 计算rho的Fisher z = 0.5*ln((1+r)/(1-r))；2) z的方差 = 1/(n-3)；3) 检测z变化的z-score =(z_t - z_{t-1}) / sqrt(2/(n-3))；4) 设定z-score>1.96(alpha=0.05)为显著性突变。这一方法有正式的统计理论支撑。 | **高** |
| S-18 | Cross-sectional sigma压缩检测：sigma<30bp且mean_rho>0.6 | **双重条件的联合分布未经校准**。sigma<30bp和mean_rho>0.6两个条件并非独立，它们的联合发生概率不能由各自的边际概率简单乘积得到。在流动性好的市场中sigma本身就小，mean_rho自然较高，两者可能在没有"标签化风险"时也经常同时满足。 | 1) 校准联合阈值：使用历史数据计算(sigma, mean_rho)的二维经验分布，在90%/95%分位数处设置联合警戒线；2) 增加**sigma与mean_rho的相关性纳入**——在样本中两者的相关性可能是负的(-0.3到-0.7)，应在检测时考虑。 | **中** |

### 2.5 压力测试模块

| # | 当前做法 | 问题所在 | 修正建议 | 优先级 |
|---|---------|---------|---------|-------|
| S-19 | 情景概率赋值：轻度20%/中度10%/严重3%/极端1% | **概率赋值缺乏校准依据**——"历史概率"来自哪个历史？基于多长时间的样本？是经济周期的"平均概率"还是包含/排除了极端时期？这些概率可能在不同市场状态下差异极大(牛市中轻度压力概率<10%，熊市中>50%)。且4个情景概率之和=34%，意味着66%的"正常"概率——但"正常"的定义并未说明。 | 1) 标注概率来源："基于YYYY-YYYY年X个样本的利差变化历史分布的经验频率"；2) 改用**条件概率**："在宏观环境X下，轻度承压概率为P1，中度P2..."；3) 增加"概率不可验证——建议视为排序性指标而非精确概率"的标注；4) 或引入**Bootstrap重抽样**从历史数据中生成情景频率分布。 | **高** |
| S-20 | 多因子情景假设因子独立，误差标注为"10-20%" | **误差范围的标注本身就是未经验证的专家估计**。真实的误差可能远大于10-20%——在系统性压力时期，因子相关性可以接近1(所有东西一起跌)，此时独立假设将低估实际影响50%以上(如2008年金融危机中各资产类别相关性从0.3跳升至0.8-0.9，联合尾部概率被低估数倍)。 | 1) 标注更保守的误差："在正常市场条件下误差约10-20%，在系统性压力时期可超过50%"；2) 引入**压力相关系数矩阵**：使用历史危机事件期间(如2020年3月、2015年股灾)的相关系数替代独立假设；3) 作为补充，使用**Cornish-Fisher展开**调整VaR估计以捕捉非正态性。 | **高** |
| S-21 | 久期近似D_approx = 待偿年数 / (1+YTM) | **该近似公式仅适用于零息债券或无票息债券**。对于付息债券(尤其是票息率远离当前YTM时)，McCaulay久期的计算需要每期现金流的加权现值，简化公式的误差随票息率与YTM之差的增大而扩大。对于票息率5%而YTM=3.5%的债券，近似久期偏误可达10-20%。 | 1) 对于有足够数据的活跃券，使用**完整的McCaulay久期公式**而非简化版：D_mac = sum(t*CF_t/(1+YTM)^t) / Bond_Price；2) 在简化版附注中标注"仅适用于票息率接近YTM的场合——偏误随票率/YTM偏离度增大"；3) 考虑在非活跃券上使用三因子久期近似(curve/level/slope)。 | **中** |

### 2.6 统计陷阱识别

| # | 问题类型 | 具体表现 | 影响评估 | 修正建议 | 
|---|---------|---------|---------|---------|
| T-1 | **幸存者偏差(Survivorship Bias)** | 实例选择的华晨汽车和双良节能均为已知的违约/信用恶化的成功预警案例。框架验证仅使用了"已知正确答案"的样本，未纳入：1) 框架产生预警信号但最终未违约的假阳性案例；2) 框架未预警但实际违约的假阴性案例。这导致框架的"验证"依赖于选择性样本。 | **高**——当前验证已从方法证明退化为故事叙述。一个指标框架的可靠性不能仅靠"它抓住了这几个知名案例"来证明。 | 1) 建立**系统性回测框架**：定义统一的信号阈值，在至少50个企业的历史数据上测试(或在可获取的范围内)；2) 报告**混淆矩阵**(TP/FP/TN/FN)及衍生的**敏感度(sensitivity)**和**特异度(specificity)**；3) 至少标注"当前案例为事后验证(ex-post)，不代表框架在实际部署中的事前(ex-ante)预测能力"。 |
| T-2 | **数据挖掘偏差(Data-snooping)** | 多个检测阈值(50bp、3sigma、30bp、delta>0.3等)可能是在分析已知案例过程中被"调优"至最佳表现。如果阈值设定参照了华晨/双良案例来"看起来准确"，那么阈值在外推样本上的有效性会显著下降。 | **中高**——没有反例或OOS(out-of-sample)验证，无法排除套用偏差。 | 1) 明确区分**训练期**(用于校准阈值的样本)和**验证期**(用于测试的hold-out样本)；2) 报告阈值在验证期的表现(而不是仅展示已看到的案例)；3) 使用**Walk-forward验证**——逐年滚动校准并测试，避免未来信息泄漏。 |
| T-3 | **小样本问题(Small Sample)** | 全文中多处使用小样本：因子回归24周(n=24但因子3-4个)、相关性矩阵(n=24)、行业利差基于N>=3家企业、尾部相关性基于N_tail=1-2。每个小样本分析的统计效力均不足以做出可靠推断。 | **高**——这是整个体系最根本的统计局限，影响因子分解、相关性分析的所有结论。 | 已在S-10、S-15、S-16中提出具体修正。全局原则：**任何统计量应标注观测值数量n和对应的置信区间**。当n<30(回归)或n<52(尾部相关)时，自动降级为"方向性参考"。 |
| T-4 | **前瞻性偏差(Look-ahead Bias)** | 历史分位数计算(如"当前利差处于90%分位数")使用的是包含整个时间段的窗口——这意味着"当前"的分位数判断受到"未来"进入样本的数据点的影响。本质上是使用全部窗口数据计算的分位数在窗口末端做实时判断的偏差。 | **中**——实时应用中使用滚动窗口时，每个时间点仅使用截至该点的数据计算分位数，不会真正产生前瞻性偏差。但确认文档中的分位数计算方式是否明确标注了这一点。 | 在滚动窗口计算中必须明确标注："分位数基于滚动窗口计算，每个时点仅使用截至该时点的历史数据，不包含未来信息"。如果文档中的实例(如双良案例)使用了事后全部数据计算分位数，则应标注"回溯分析，与实时分析的计算方式不同"。 |
| T-5 | **多重比较问题(Multiple Comparisons)** | 框架同时监控多个信号类型、多家企业、多个阈值条件。当同时监控100家企业*5种检测规则=500次检测时，即使所有企业都"正常"(H0为真)，预期也会有约25个(5%显著性水平)假阳性。但框架目前没有对多重比较做任何校正。 | **高**——在监控较多样本时，假阳性累积效应会显著降低信号的可信度。一个实际部署的系统可能在每个报告期产生大量"红色警报"而大部分是误报。 | 1) 引入**Benjamini-Hochberg FDR校正**(较Bonferroni更灵敏)；2) 在报告中同时输出**总检测次数N**和**发现的信号数k**，并为每次信号标注"经FDR校正"或"未经校正"；3) 增加**信号一致性要求**——只有≥2个独立检测模块同时触发时才判定为确认信号(降低单模块误报影响)。 |

## 第三部分：未覆盖的统计方法建议

以下方法当前版本未覆盖，但可显著增强定量分析的有效性。按实施优先级排列。

### 3.1 高优先级(建议尽快整合)

| 方法 | 用途 | 适用模块 | 预期效果 |
|------|------|---------|---------|
| **Bootstrap重抽样** | 为所有统计量(均值/中位数/相关系数/回归系数/波动率)生成置信区间和标准误，不依赖正态假设 | 全部 | 以前: 点估计->不可知不确定性; 以后: "beta=0.42, 90%CI=[0.18,0.66]"——统计推断的基石 |
| **Fisher z变换与相关系数推断** | 为Pearson相关系数构建置信区间、检验相关系数是否显著、检验两个相关系数是否显著不同 | 相关性分析(Section 5) | 替代"delta>0.3"这个arbitrary阈值，使用假设检验框架判断相关性变化是否统计显著 |
| **ADF平稳性检验** | 检测利差序列和因子序列是否非平稳，避免伪回归 | 多因子回归(Section 4) | 防止两个无关的随机游走序列产生貌似显著的回归关系(伪回归R²可>0.5) |
| **Newey-West异方差自相关一致标准误** | 在存在异方差和自相关时获得正确的标准误估计 | 多因子回归(Section 4) | OLS的标准误在存在异方差时偏小->beta的t统计量偏大->虚假显著。NW标准误纠正此偏误 |

### 3.2 中优先级(视数据丰富度渐进整合)

| 方法 | 用途 | 适用模块 | 预期效果 |
|------|------|---------|---------|
| **Granger因果检验** | 验证股价变化/波动率变化对信用利差是否存在统计上显著的预测能力(领先关系) | 市场隐含信号(Section 7) | 当前"股价领先债市2-12周"是专家估计。Granger检验可在给定滞后期(如4周)下判断预测能力是否统计显著 |
| **协整检验(Engle-Granger/Johansen)** | 检验两个或多个非平稳利差序列是否存在长期均衡关系(当被违背时即为套利机会或风险信号) | 利差分析(Section 2)、相关性(Section 5) | 协整关系被破坏(残差远离均值)可能是定价失真或信用分化的早期信号，比Pearson相关系数更适用于非平稳序列 |
| **Chow断点检验/Bai-Perron多重结构断点** | 在时间序列中检测回归系数的结构性变化(无需主观选择是否分前后两段) | 因子时变(Section 4.3) | 替代"检查beta趋势"的主观判断——自动检测beta是否存在统计显著的断点，并确定最可能的分割点 |
| **GARCH(1,1)波动率模型** | 建模波动率聚集性：高波动后倾向于继续高波动，低波动后倾向于继续低波动 | 波动率分析(Section 3) | HV是对过去的简单平均，GARCH能预测下一期波动率。GARCH预测的波动率可作为波动率突变的前瞻性信号 |
| **Diebold-Mariano检验** | 比较不同波动率模型(HV_20d vs GARCH vs EWMA)的预测能力是否有统计显著差异 | 波动率分析(Section 3) | 判断"哪个窗口/模型更好"从主观选择变为统计检验 |

### 3.3 低优先级(长期优化)

| 方法 | 用途 | 预期收益 | 依赖条件 |
|------|------|---------|---------|
| **Copula模型(Clayton/Gumbel/t-Copula)** | 建模尾部相关性(非线性非对称依赖) | 远超当前"5%分位数同跌比例"的粗糙处理；可量化上尾/下尾依赖的不对称性 | 需要足够面板数据(至少3年周度数据, N>=150) |
| **极值理论(EVT) / Peak-Over-Threshold(POT)** | 用广义帕累托分布(GPD)拟合超过高阈值的极端损失 | 更准确的压力情景概率估值——替代当前arbitrary的情景概率赋值 | 需要足够多的极端样本(至少50个极端值的POT分析才可靠) |
| **Kalman滤波(时变参数模型)** | beta/alpha的动态时变估计——一个观测值到达即更新一次参数 | 替代滚动窗口的"块状"beta估计，参数估计更平滑、反应更快 | 计算复杂度增加，需要更成熟的工程实现 |
| **Lasso/Ridge/Elastic Net正则化回归** | 在多重共线性和小样本下实现因子选择和收缩 | 因子筛选更稳定——尤其在观测值维度远小于因子数时 | 当因子数增加至>5个且n<30时才有显著优势 |
| **贝叶斯结构时间序列模型(BSTS)** | 将因果推断纳入：某个事件(如评级下调)对利差产生了多少因果效应 | 可以提供"评级下调导致利差走阔Xbp"的因果估计而非仅描述相关性 | 实现复杂度高，需要明确的事件时间和足够的反事实观测 |

## 第四部分：统计最佳实践清单

以下清单可作为未来开发和代码实现的检查清单。每一项都标注了出处(当前文档已覆盖/本审计建议新增)。

### 4.1 数据前处理

- [ ] **缺失值处理**: 明确标注缺失机制(随机缺失/非随机缺失)及插补方法(线性插值/最后一次观测推进法LOCF/多重插补MI)
  - (当前: 有质量标注但未明确缺失处理规则)
- [ ] **异常值诊断**: 区分极端值(genuine extreme)和异常值(outlier/数据错误)，使用IQR规则或Hampel滤波器诊断
  - (当前: 有一个基本框架，但缺失系统性误标区分)
- [x] **平稳性检验**: 任何时间序列回归前执行ADF或KPSS检验
  - (新增: S-11)
- [ ] **数据对齐**: 所有时间序列在同一频率下对齐(如从日度到周度)，明确对齐策略(取周期末/取周期均值)
  - (当前: 部分覆盖，但周度数据从日度聚合的方法未指定)

### 4.2 参数与模型选择

- [ ] **窗口选择的原则**: 每个窗口参数(20/24/60/120日)标注选择理由——业务理由vs统计理由vs惯例
  - (当前: 部分标注如"≈1个月/≈1季度"，但统计理由不足)
- [x] **多窗口并行输出**: 使用至少两种窗口同时分析，以趋势方向而非绝对值为准
  - (当前: 波动率三窗并行是良好实践)
- [ ] **最小样本量规则**: 对每种统计方法明确标注最小观测值阈值，低于阈值时自动降级
  - (当前: 因子回归标注12周最小值，但此阈值过低。新增: S-10建议提升至30)
- [x] **正则化/收缩**: 小样本回归时使用Lasso/贝叶斯方法控制过拟合
  - (新增: S-10)
- [ ] **模型比较基准**: 任何回归模型应与一个朴素基准(如历史均值)比较，不仅看R²/AIC
  - (当前: 未覆盖)

### 4.3 统计推断

- [x] **置信区间而非点估计**: 所有统计量(均值/中位数/相关系数/回归系数/波动率)报告置信区间
  - (新增: Bootstrap, Fisher z — S-10, S-15)
- [x] **p值+效应量**: 同时报告统计显著性(p值)和实际显著性(效应量/Cohen's d/相关系数本身)
  - (新增: S-15)
- [ ] **假设检验的前提验证**: 每个检验前验证其统计前提，前提不满足时使用非参数替代
  - (当前: 部分覆盖。新增: S-11平稳性, S-12异方差)
- [x] **多重比较校正**: 同时运行多个检测时使用FDR控制(Benjamini-Hochberg)
  - (新增: S-2, T-5)
- [ ] **回测/事后验证**: 任何阈值或规则在新部署前应在历史数据上做回测，报告TPR/SPR/PPV
  - (新增: T-1, T-2)

### 4.4 时间序列专项

- [ ] **自相关诊断**: 回归残差做Ljung-Box检验或Durbin-Watson检验(当前已覆盖Durbin-Watson，建议增加Ljung-Box)
  - (当前: 部分覆盖)
- [x] **异方差处理**: 使用Breusch-Pagan检验诊断，使用Newey-West标准误修正
  - (新增: S-12)
- [ ] **领先-滞后分析**: 使用交叉相关函数(CCF)或Granger因果检验判断变量间的领先关系
  - (新增: Granger因果)
- [x] **结构断点检测**: 使用Chow检验或Bai-Perron方法检测参数的结构性变化
  - (新增: S-14)
- [ ] **波动率建模**: 考虑使用EWMA或GARCH(1,1)作为HV的补充
  - (新增: S-6)

### 4.5 相关性专项

- [x] **相关系数的Fisher z置信区间**: 每次输出相关系数时附上置信区间和显著性检验
  - (新增: S-15, Fisher z变换)
- [x] **非参数相关作为稳健性检查**: 使用Spearman's rho作为Pearson的稳健性替代
  - (建议新增——中优先级)
- [x] **相关性差异的统计检验**: 使用Fisher z检验判断相关系数是否显著不同
  - (新增: S-17)
- [x] **尾部相关性的样本量问题标注**: 当尾部有效样本<5时，标注"不可靠"
  - (新增: S-16)

### 4.6 压力测试专项

- [ ] **概率校准**: 情景概率应来自历史经验的、经过统计检验的定量校准，而非专家判断
  - (新增: S-19)
- [x] **假设的敏感性分析**: 每个关键假设(如因子独立)应做敏感性分析，展示假设违反时结论如何变化
  - (新增: S-20——压力时期因子相关性上升的影响)
- [ ] **Bootstrap压力情景**: 从历史数据中有放回地重抽样生成情景分布，作为蒙特卡洛的轻度替代
  - (新增: 中优先级建议)
- [ ] **逆向压力测试的统计框架**: "什么情景会导致违约?"的问题应使用分位数回归或极值理论，而非主观构造
  - (当前: 基本框架ok，但缺乏统计方法支撑)

### 4.7 输出与报告

- [x] **质量标注**: 每项指标标注数据质量等级、来源、窗口
  - (当前: 已覆盖，是全局最佳)
- [ ] **不确定性表达**: 任何量化结论附带不确定性度量(置信区间/标准误/分位数范围)
  - (新增: 全局改进)
- [x] **替代方案标注**: 当精确指标不可获取时标注替代方案和预期误差
  - (当前: 已覆盖，良好)
- [ ] **向上沟通规则**: 避免因果语言——报告"YTM利差与行业利差中位数呈正相关"，而非"行业利差上升导致个券利差上升"
  - (当前: 部分覆盖需强化)
- [ ] **Type I/Type II错误率**: 每个检测规则的阈值选择应报告其历史假阳性率和假阴性率(经回测)
  - (当前: 未覆盖)

### 4.8 方法论完整性

- [ ] **样本外验证**: 任何结论(在实例/案例中展示的)应至少在hold-out样本上验证一次
  - (新增: T-2)
- [ ] **前向验证**: 确定规则后，用之后的数据做前向测试(避免未来信息)
  - (新增: T-4)

### 5.3 最关键的三项单一改进

如果只做三项改进就能最大程度提升统计严谨性：

1. **为所有统计量附加置信区间** (Bootstrap + Fisher z) —— 消除了"点估计"带来的虚假精确性幻觉，所有推断自带不确定性度量。

2. **增加平稳性检验 + 异方差校正** (ADF + Newey-West) —— 防止最严重的统计错误：伪回归和虚假显著性。没有这两个，因子回归的所有结论都可能不可靠。

3. **引入多重比较FDR校正** (Benjamini-Hochberg) —— 实际部署中最常见的问题：信号太多(大部分是误报)导致使用者不再信任所有信号。FDR校正是维持信号可信度的关键防线。
