### 🔴P0-8：SRI指标在预警框架与输出框架之间的量纲不一致

- **文件**: `dev/engine/systemic-warning-framework.md` §三（第300-307行）vs `dev/engine/output-layered-framework.md` §3.6（第208-243行）
- **问题**: systemic-warning-framework.md定义SRI范围为0-3+（🟢<0.5, 🟡0.5-1.0, 🟠1.0-1.8, 🔴≥1.8），当前SRI=0.56为🟡。但output-layered-framework.md §3.6的温度计卡片示例使用了完全不同的0-100分制：
  - 示例🟡关注卡片："SRI: 38/100"
  - 示例🔴危险卡片："SRI: 78/100"
  - 示例🟢风险解除："SRI: 22/100"
  如果0.56映射到56/100，38/100理论上应是🟢（<0.5），78/100应是🟠（0.78<1.0），与示例的颜色标注矛盾。两套量纲完全不可互转。
- **严重度**: 🔴P0 — 预警框架和输出框架对同一指标使用不同量纲，输出层示例与理论层定义直接冲突
- **修复建议**: 统一量纲。建议输出框架使用与理论框架一致的0-3+范围（SRI: 0.56），或明确约定百分比映射规则（如SRI% = SRI×100/3，则0.56→18.7/100，需同步更新所有示例数值）

### 🔴P0-9：SRI计算公式与计算实例不一致

- **文件**: `dev/engine/systemic-warning-framework.md` §2.2（第123-156行）
- **问题**: §2.2定义的公式为：
  ```
  SRI = Σ(行业风险得分 × 行业权重) / 13
  ```
  其中"行业权重在各行业间归一化，确保Σ(行业权重)=13"。但在§8.3（第817-826行）的实际计算中，行业权重使用百分比形式且未除以13：
  ```
  城投债(25%权重): 得1.5分 × 25% = 0.375
  SRI = 0.375 + 0.058 + 0.044 + 0.080 = 0.557
  ```
  如果公式正确（除以13），则SRI应为0.557/13=0.043；如果百分比权重直接使用正确，则公式应改为SRI=Σ(risk×pct%)。
- **严重度**: 🔴P0 — 核心公式与3个回测+1个当前计算的实现方式不一致，算法无法复现
- **修复建议**: 根据实际计算方式修正公式为`SRI = Σ(行业风险得分 × 行业权重百分比)`，或保留原公式但确保归一化解释清晰（权重归一化至13后除以13等价于使用百分比权重直接加权）。同时，统一所有回测和当前计算的表达方式
