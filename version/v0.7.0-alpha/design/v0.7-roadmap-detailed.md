## 阶段一：跨行业传染图谱（v0.6.1 → v0.6.4）

### v0.6.1：传染理论基础

**执行指导**：
1. 创建 `dev/engine/contagion-theory.md`
2. 定义四种传染类型：信用链传染（上下游应收账款/担保链）·区域共振（同一地方政府信用背书）·流动性挤兑（同一融资渠道/债券品种）·信心崩塌（市场叙事驱动的非理性传染）
3. 为每种类型建立：触发条件·传导速度·历史振幅·衰减距离·可预测性评分
4. 定义7条标准传导路径模板：供应商-客户链·担保圈·同区域国企·同评级·同行业·同融资渠道·同实际控制人
5. 建立传染路径与现有6种分析范式的映射关系——每种范式下哪些传染类型最危险

**验收标准**：
- 四种传染类型各有明确的触发条件+传导速度+衰减距离
- 7条传导路径各有至少1个真实历史案例佐证
- 与6种范式的映射表完整
- 文档 >= 600行

**验收标准**：
- 169个单元格(13×13)全部填充

## 阶段二：组合集中度仪表盘（v0.6.5 → v0.6.7）

### v0.6.5：五维集中度框架

**执行指导**：
1. 创建 `dev/engine/concentration-framework.md`
2. 五维各定义：
   - **行业集中度**：HHI指数·前3行业占比·单一行业上限阈值
   - **区域集中度**：省份/城市级别分类·同一区域发行人相关性系数·区域财政健康度加权
   - **评级集中度**：外部评级vs内部评级的集中度差异（"伪高评级"集中风险）
   - **期限集中度**：到期时间分布·未来12/24/36个月的集中到期比例
   - **融资渠道集中度**：银行/债券/非标/租赁·单一渠道依赖度
3. 每维定义：绿/黄/橙/红四级阈值·超阈值→评级调整幅度·调整规则的理由
4. 定义五维加权综合评分公式

**验收标准**：
- 五维各有独立阈值表
- 每个阈值有理由说明(引用历史数据或理论依据)
- 综合评分公式明确
- 文档 >= 500行

## 阶段三：系统性预警体系（v0.6.8 → v0.6.9）

### v0.6.8：信号聚合算法+温度计

**执行指导**：
1. 创建 `dev/engine/systemic-warning-framework.md`
2. 信号聚合规则：
   - 输入：13行业的轨道A评级+轨道B信号+展望方向
   - 聚合：统计同时处于"负面展望"或"高风险"评级的行业数量
   - 加权：行业体量(信用债存量)越大→权重越高·传染力越强的行业→权重越高
3. 温度计四级：
   - 🟢 正常(<20%行业同时红色·或>70%行业绿色)
   - 🟡 关注(20-30%行业同时红色)
   - 🟠 警惕(30-50%行业同时红色·或高传染行业出问题)
   - 🔴 危险(>50%行业同时红色·或多个高传染行业同时触发)
4. 用永煤(2020Q4)、地产(2021Q3)、疫情(2020Q1)三个时点做回测——温度计是否能提前3-6个月给出预警？
5. WebSearch获取13行业2026Q2当前信号数据，计算当前温度

**验收标准**：
- 信号聚合算法有明确的公式和阈值
- 3次历史回测全部完成·至少2次提前≥3个月预警
- 当前温度计算完成
- 文档 >= 600行
