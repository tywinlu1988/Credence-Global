# v0.8.0 集成发布 — 8 条 active 工作路径端到端走查记录

**报告版本**: v1.0 | **对应引擎版本**: v0.8.0-release | **日期**: 2026-07-16

> **本文档性质**：这是 **v0.8.0 集成发布** 的端到端走查证据（integration evidence），属于
> **append-only 测试输出**，存档于 `validation/docs/`。它**不是项目本体**，永不进入 `version/`
> 快照。每条走查把一条 active 工作路径从"模拟业务需求"一路走到"路由四问 → 工作路径单 →
> analysis → report → qa"，验证 v0.7.6–v0.7.9 阶梯交付的四段链（intake/analysis/report/qa）
> 在集成点上可端到端走通。
>
> **单一事实源声明**：本记录只展示引擎/编排器**真实产出的读数**（两条已接线路径的代码运行
> 结果），**不复制**任何阈值、权重、评级映射或分层时间预算。路径单中的 `engine_reading_order`
> 与 `quality_gates` 原样取自 [work-path-registry.md](../../dev/engine/work-path-registry.md)；
> 阶段名/承载 skill 原样取自 [pipeline-contract.md](../../dev/engine/pipeline-contract.md)。
> 凡涉及数值语义，一律以所引引擎文档章节为最终裁决。
>
> **走查方法**：对每条路径，用 `src/path_sheet.py.load_registry_paths` 读注册表构造合法路径单，
> 用 `src/pipeline.py.load_stage_plan` 生成四段计划，再用 `run_executable_stages` 执行。两条
> **已接线（wired）** 路径（WP-M4-01 → 五维集中度、WP-M4-03 → SRI）的 analysis 阶段以
> `mode="code"` 真实运行编码引擎并粘贴其真实输出；其余 6 条为 **LLM 编排** 路径，各阶段标记
> `llm-orchestrated`（analysis 阶段 `executable=False`，由 LLM 按引擎文档编排，优雅跳过）。
